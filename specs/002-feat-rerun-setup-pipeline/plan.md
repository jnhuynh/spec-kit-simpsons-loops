# Implementation Plan: Rerunnable Setup & End-to-End Pipeline

**Branch**: `002-feat-rerun-setup-pipeline` | **Date**: 2026-03-09 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/002-feat-rerun-setup-pipeline/spec.md`

## Summary

Make `setup.sh` safely rerunnable by extracting quality gate configuration into a dedicated file (`.specify/quality-gates.sh`) that is never overwritten on subsequent runs. Update all loop scripts (`ralph-loop.sh`, `pipeline.sh`) and command files to read quality gates from this file as the default source, with CLI and env var overrides preserved. Extend the pipeline with an optional "specify" step that creates a spec from a feature description non-interactively, enabling true end-to-end automation from idea to implementation.

## Technical Context

**Language/Version**: Bash 4+ (shell scripts), Markdown (command/agent files)
**Primary Dependencies**: Claude CLI (`claude` command), `jq` (optional, for settings updates), standard Unix utilities (`grep`, `sed`, `awk`, `mktemp`, `mv`, `chmod`)
**Storage**: Filesystem only — shell scripts, markdown files, and configuration files
**Testing**: Manual validation via `shellcheck` for linting, functional testing by running `setup.sh` and `pipeline.sh` against test projects
**Target Platform**: Linux, macOS (any system with bash 4+)
**Project Type**: CLI toolkit / developer tooling (shell scripts + markdown command files)
**Performance Goals**: N/A (developer tooling, not performance-critical)
**Constraints**: Must be idempotent; must not corrupt existing configuration; must work without `jq` (graceful degradation)
**Scale/Scope**: ~10 shell scripts, ~10 markdown command/agent files; changes touch 6-8 files
